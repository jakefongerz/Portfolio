unitConv API

has the following parameters...

  • from is a string which identifies the type of units the input value represents
  • value is a non-negative floating-point number representing the number of from units
  • to is a string representing the type of units the user wants value converted into

a Django model holds all oof the conversion factors. Create a data migration to initialize the database with the necessary unit conversion factors. This is the only place in your program where you are allowed to hard-code the conversion factors. The API must be capable of converting between all supported units

javascript

shows only expressions with undefined results. If there are no undefined expressions. Make an asynchronous GET request using fetch(), requesting the data in JSON format. At minimum the request must include your API key and a limiting range. By default, Nasdaq Data Link responds with all of the data it has since the beginning of its records. Study the documentation to learn how to request only the most recent data point. Upon receipt of the request, extract the price in USD per troy ounce of gold from the JSON payload and store it in a global variable so it is available for use when the user clicks the button.This page only shows a title and the navigation links
